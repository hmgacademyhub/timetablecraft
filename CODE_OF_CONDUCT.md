# Code of Conduct

The **TimetableCraft** project is a part of the **HMG Concepts** family.
Our entire organisation is built on one philosophy:

> *"Learning Deliberately. Teaching Authentically."*

That philosophy guides this project's community too.

---

## Our standards

We expect everyone — contributors, maintainers, users, students, teachers, partner schools — to:

- ✅ Be **kind**. Assume good intent. Default to generosity.
- ✅ Be **patient**. Many of our contributors are educators learning to code, or developers learning the Nigerian classroom context. Both are valuable.
- ✅ Be **specific**. When you give feedback, point to the line, the file, the workflow. Vague criticism helps nobody.
- ✅ Be **professional**. This tool helps real schools run real timetables. Treat the codebase, the issue tracker, and each other with that level of seriousness.
- ✅ Credit work. If someone helped you, say so. Especially the teachers whose lived experience shapes every feature.

We will **not tolerate**:

- ❌ Harassment, discrimination, or personal attacks of any kind.
- ❌ Tribal, ethnic, religious, gender, or political insults — Nigerian or otherwise.
- ❌ Sexual language or imagery in any project space.
- ❌ Disclosing private contributor details without consent.
- ❌ Aggressive, demanding behaviour against maintainers who are volunteering their time.

---

## Scope

This Code of Conduct applies everywhere the project lives — GitHub issues & PRs, the live Streamlit app, the project WhatsApp / LinkedIn channels, online or in-person events, and any HMG Academy event where TimetableCraft is demonstrated.

## Enforcement

Maintainers may, at their discretion:

1. Issue a private warning.
2. Issue a public warning on the relevant issue/PR.
3. Temporarily block the user from the project.
4. Permanently block the user from the project.
